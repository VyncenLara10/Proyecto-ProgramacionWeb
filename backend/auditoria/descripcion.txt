APP: auditoria

Esta app garantiza la trazabilidad completa del sistema y cumple con el principio de no repudio.
Registra toda acción relevante de usuarios y administradores, y documenta todos los correos enviados por la aplicación.
También valida la identidad de los usuarios mediante los tokens JWT recibidos en cada acción registrada.

FUNCIONALIDAD PRINCIPAL:
- Mantener una bitácora general de eventos (quién hizo qué, cuándo y desde dónde).
- Registrar acciones como:
  - Registro o inicio de sesión de usuario.
  - Habilitación o deshabilitación.
  - Compras, ventas o movimientos.
  - Generación o envío de reportes.
- Guardar información de contexto (IP, navegador, user agent).
- Registrar todos los correos enviados por el sistema, su tipo, estado y errores.
- Asociar cada registro de acción con el `user_id` proveniente del token JWT.
- Integrarse con las demás apps para recibir eventos automáticos.

MODELOS PRINCIPALES:
- Bitacora → Registra todas las acciones del sistema con usuario, acción, fecha, IP y origen.
- CorreoLog → Registra los correos enviados (tipo, asunto, contenido, estado, error).

JWT:
- Cada acción registrada en la bitácora debe asociarse con el usuario autenticado identificado por JWT.
- Si la acción proviene de un token inválido o expirado, se rechaza el registro.

NO REPUDIO:
- Cada evento crítico genera una entrada verificable en la bitácora con el usuario autenticado.
- Cada correo queda registrado con hora, usuario y estado de envío.
- Los tokens JWT son evidencia digital de la identidad de quien realizó la acción.
- Permite reconstruir todo el historial de acciones, comunicaciones y autenticaciones.
